Библиотеки для валидации форм (по удобству работы с типами):
1) Zod (типы выводятся автоматически из схемы);
2) Valibot (то же самое, но легче и быстрее);
3) Yup (больше ручных действий);
4) class-validator (типы получаются только из класса, а не из схемы валидации);
5) Joi (самая старая, но мощная).

Точка входа: import JustValidate from 'just-validate';
Основные методы/функции/классы для работы с валидацией формы:
1) JustValidate (Основной класс-валидатор. Создаётся экземпляр для формы);
2) addField (Добавляет поле(я) для валидации);
3) addRule (Регистрирует кастомное правило валидации);
4) validate() (Запускает валидацию всех полей);
5) onSuccess(callback) (Устанавливает колбэк на успешную валидацию);
6) onFail(callback) (Устанавливает колбэк на ошибку валидации);
7) setCurrentError(field, error) (Устанавливает кастомную ошибку для поля).
Минимальный набор для работы с валидацией форм:
1) JustValidate — создание экземпляра;
2) addField — добавление полей с правилами;
3) validate() или хуки (onSuccess/onFail) — запуск и обработка.
Методы, которые могут чаще всего понадобиться:
1) addField(selector, rules) — подключение поля и описания правил (required, email, minLength и др.).
2) onSuccess((event) => {...}) и onFail((fields) => {...}) — реакции на успешную и неудачную валидацию (отправка, показ ошибок).
3) refresh() — ручной перезапуск валидации, если данные формы менялись динамически.
